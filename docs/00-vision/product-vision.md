# AgentOS — 产品愿景

## 一句话定义

**一个常驻在 Android 系统中、可随时呼出的智能私人助理：理解你说的话，动手帮你做事。**

## 想解决的问题

用户在日常吃穿住行娱乐场景里，被各 App 的入口、流程、广告、对话框反复消耗：

- 想看一部剧 → 不知道哪个平台有 → 切应用搜索 → 选剧 → 点续看
- 看到喜欢的衣服 → 截图 → 切应用 → 拍照搜 → 比价 → 下单 → 付款
- 点外卖 → 翻历史订单 → 改地址 → 选时间 → 凑满减 → 付款
- 打车 → 输起点终点 → 等价 → 比平台 → 叫车

每一次都是一连串机械的、重复的、毫无价值的点击和等待。

## 我们提供什么

> "我想看三体" → 助手自动跳转腾讯视频，搜剧，从上次进度继续播放。
>
> "我想买一件刘亦菲同款" → 助手自动比价，跳到京东对应商品，停在付款页等用户输入密码。
>
> 全程无需打开多个 App，无需重复输入。

## 北极星指标

- **每次完成事务的人工点击数 ≤ 2**（首次确认 + 付款确认）
- **重复场景的 token 消耗 ≤ 首次的 10%**（Skill 缓存命中）
- **用户首次激活后 30 天留存 ≥ 60%**

## 用户画像

1. **效率型用户**：嫌弃 App 切换繁琐的中青年用户（目标主力）
2. **行动不便用户**：老人 / 视力障碍者（无障碍场景天然契合）
3. **重度场景用户**：经常点外卖 / 打车 / 看视频 / 购物的人

## 不做什么

- 不做 chatbot：不是聊天产品，是动作产品
- 不做"我能不能问个事儿"：是"帮我把这事儿干了"
- 不做后台静默：所有动作用户能看到、能打断、能撤销

## 与同类产品的边界

| 维度 | AgentOS | 系统语音助手 (小爱 / 小艺) | 自动化工具 (Tasker / 自动点击器) |
|---|---|---|---|
| 跨 App 操控 | ✅ 自然语言驱动 | △ 仅厂商生态内 | △ 需用户写脚本 |
| 学习用户习惯 | ✅ Skill + 偏好画像 | △ | ✕ |
| 商用上架 | ✅ AccessibilityService | ✅ 系统级 | △ 部分上架 |
| 用户上手 | 一次性配置后零学习成本 | 仅命令式语音 | 需懂自动化 |

## 道德 / 隐私边界

- 不可逆操作（付款、删除、发送、外联消息）必须用户显式确认
- 屏幕内容默认只传摘要给 LLM，用户可关闭云端
- 用户可随时清空所有学习到的偏好与历史
