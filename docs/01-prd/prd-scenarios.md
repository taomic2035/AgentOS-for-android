# PRD — 五大场景脚本

> 每个场景给出：触发语 → 期望流程 → 守卫点 → 验收。

## 场景 1：看视频

**触发语**：
- "我想看三体"
- "三体最新一集"
- "继续看三体"

**期望流程**：

1. Agent 识别意图 = 播放剧集，剧名 = "三体"
2. 查询本地偏好库：用户之前在哪个平台看过？→ 命中"腾讯视频"
3. 命中 Skill `tencent_video_play(title=三体)`
4. 拉起腾讯视频 → 搜索 "三体" → 选择对应剧集 → 自动续看（如平台支持）

**守卫点**：

- 若需登录 → 停在登录页让用户操作
- 若是付费内容且未购买 → 停在购买页等待用户确认

**验收**：

- 端到端 ≤ 5 秒（不含 App 冷启动）
- Skill 命中后 token 消耗 = 0（不调 LLM）

## 场景 2：购物（同款搜索）

**触发语**：
- "我也想买一件刘亦菲同款衣服"
- "刚才那件衣服在京东多少钱"

**期望流程**：

1. Agent 截图 / 读屏 → 识别衣服关键特征（颜色 / 款式 / 演员名）
2. LLM 生成搜索 query
3. 命中 Skill `shopping_jd_search_buy(query=...)`
4. 拉起京东 → 搜索 → 比价 / 选销量第一 → 进入商品页 → 选择规格 → 下单 → **停在付款页等用户输密码**

**守卫点**：

- 商品超过 ¥500 必弹"金额确认"
- 付款页面前必停下，由用户输密码 / 指纹

**验收**：

- 用户操作 ≤ 2 次（确认 + 付款）
- 付款页面前 100% 停下

## 场景 3：外卖

**触发语**：
- "中午老地方那家"
- "饿了，订个炒饭"

**期望流程**：

1. 命中"老地方" → 偏好库查询用户最近 30 天最常下单的店铺
2. 命中 Skill `meituan_reorder(shop_id=...)` 或 `eleme_reorder(shop_id=...)`
3. 直接进入店铺 → 用历史订单的同款 → 选默认地址 → 选默认时间 → 停在付款页

**守卫点**：

- 若历史订单不存在 → 反问澄清
- 总价异常波动（>+30% 历史均值）必弹确认

**验收**：

- 用户操作 ≤ 2 次

## 场景 4：打车

**触发语**：
- "回家"
- "打车去公司"

**期望流程**：

1. 命中"回家" / "公司" → 偏好库查询地址
2. 命中 Skill `didi_call(from=current, to=home_address)`
3. 拉起滴滴 / 高德 → 自动填地址 → 选默认车型 → **停在叫车确认页**

**守卫点**：

- 价格 > 100 必弹确认
- 时段异常（深夜）必弹确认

**验收**：

- 用户操作 ≤ 1 次（叫车确认）

---

## 场景 5：搜索

**触发语**：
- "查一下天气"
- "搜小米 17 评测"
- "微信搜一搜 ChatGPT 最新动态"

**期望流程**：

1. Agent 识别搜索词 + 优先平台
2. 命中 Skill `search_*` 或直接 deep link
3. 拉起搜索结果页

**守卫点**：

- 无（搜索是只读操作）

**验收**：

- 端到端 ≤ 3 秒
