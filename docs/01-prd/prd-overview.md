# PRD — 总览

## 目的

定义 AgentOS 在 V1.0 商用版的全功能边界、用户角色、核心场景与非功能要求，作为整个团队的需求真相源。

## 范围

- V1.0 全部用户可见功能
- 不含运营 / 增长 / 商业化策略

## 用户角色

| 角色 | 描述 | 占比 |
|---|---|---|
| 个人效率用户 | 嫌弃 App 切换繁琐的中青年 | ~70% |
| 行动不便用户 | 老人 / 视障 / 临时单手 | ~15% |
| 重度场景用户 | 频繁外卖 / 打车 / 视频 / 购物 | ~15% |

## 核心需求

- 随时呼出（浮窗 / 通知 Tile / 长按 Home / 系统手势）
- 输入：语音 + 文字双闭环
- 理解：自然语言意图识别
- 执行：跨 App 自动操作（含 Skill 缓存机制）
- 学习：记录习惯、抽取偏好、复用过往结果
- 守卫：不可逆操作前置确认
- 配置：LLM 自定义 / 隐私开关 / Skill 管理

## 接口

PRD 输出消费方：

- 架构文档：依据 PRD 设计模块
- 测试文档：依据 PRD 设计测试矩阵
- 商店上架素材：依据 PRD 写文案

## 风险

- 国内厂商 ROM 杀后台，可能影响"常驻"
- App 升级导致 Skill 失效
- 用户对 A11y 权限的接受度

## 验收

- 所有功能项在 `prd-feature-list.md` 中可追溯到一个或多个版本
- 五大场景在 `prd-scenarios.md` 中有可执行脚本
- 非功能需求在 `prd-non-functional.md` 中有可量化指标
