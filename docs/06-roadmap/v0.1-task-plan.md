# V0.1 任务计划（调整后）

> 本文档为 V0.1 实施阶段的真相源。基于 2026-04-26 "迭代前反思三问" 调整。
> 原顺序见 [`/Users/taomic/.claude/plans/eventual-tumbling-perlis.md`](../../) §6。

## 调整决策（来自三问反思）

| 项 | 调整前 | 调整后 | 理由 |
|---|---|---|---|
| 顺序 | 自顶向下：UI → Service → 浮窗 → A11y → SkillRunner → E2E | **垂直切片，风险前置** | 最大风险（A11y / 三方 App 兼容）必须早暴露 |
| 新增 T-000 | — | 模拟器 + A11y 可行性验证 | 商用项目地基不稳，后面全推倒重来 |
| 新增 T-000b | — | 自建 stub-video App | E2E / CI 不依赖三方 App 改版才能稳 |
| SkillRunner 接口 | 仅满足硬编码 | **按 V0.2 LLM tool_call 契约设计** | 避免 V0.2 重构 |

## 关键风险预判与缓解

| 风险 | 缓解策略 |
|---|---|
| 腾讯视频不能在 ARM64 模拟器上运行 | T-000c 的 stub-video 兜底；E2E 默认对 stub 跑；真实 App 作为可选环节 |
| Pixel 6 ARM64 模拟器 A11y 限制 | T-000b 早期验证；如有问题切换到真机或调整 AVD 配置 |
| AGP 8.2 / Kotlin 1.9 与 Compose BOM 2024.02 兼容 | 已在骨架阶段验证 `./gradlew projects` 通过；首次 assembleDebug 时复查 |
| Doubao 默认 base URL 国内可达性 | 仅 V0.2 才接入 LLM；V0.1 不依赖网络 |
| 模拟器冷启动慢 | 脚本里加 `-no-snapshot-load`；CI 跑 nightly 而非 PR |

## SkillRunner 接口契约（前瞻设计）

```kotlin
// core 模块（接口）
data class SkillResult(
    val ok: Boolean,
    val stepsExecuted: Int,
    val totalSteps: Int,
    val error: AgentError? = null,
    val log: List<String> = emptyList(),
)

interface SkillRunner {
    suspend fun run(
        skillId: String,
        inputs: Map<String, Any?> = emptyMap(),
    ): SkillResult
}

// V0.2 LLM tool_call 直接对应：
//   { "name": "run_skill",
//     "arguments": { "skill_id": "tencent_video_play",
//                    "inputs": { "title": "三体" } } }
// 解析 arguments → SkillRunner.run(skill_id, inputs) 零适配
```
