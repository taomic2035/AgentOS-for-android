# 流程：付款与不可逆操作确认

## 不可逆操作分类

- 付款 / 转账
- 删除（订单 / 文件 / 联系人）
- 发送（消息 / 邮件 / 评论）
- 取消订单 / 退款
- 任何带"不可恢复"语义的动作

## 检测策略（V0.7）

### 静态规则

- DSL 步骤声明 `risk: payment | delete | send`
- 内置敏感关键词列表（中英双语）：付款 / 提交订单 / 立即支付 / 删除 / 确认发送 / pay / submit / delete / send
- 节点 text / desc 命中关键词 → 触发守卫

### 动态识别（V0.8）

- 屏幕语义里识别金额（正则 `¥\d+(\.\d+)?`）
- 识别"密码框" `class=android.widget.EditText + inputType=password`
- 识别"指纹"" Face Unlock"提示

## 守卫流程

```
DSL 步骤 ──before execute──> RiskGuard.check
                                  │
                                  ├─ 不命中 ──> 继续执行
                                  │
                                  └─ 命中 ──> 暂停
                                              │
                                              ▼
                                       UI 弹"确认操作"对话框
                                       （含截图 / 金额 / 步骤说明）
                                              │
                                  ┌───────────┴───────────┐
                              用户确认                  用户取消
                                  │                       │
                                  ▼                       ▼
                              继续执行                 终止任务
```

## 确认 UI 设计

- 浮窗放大为半屏对话框
- 显示：
  - 当前屏幕截图缩略
  - 即将执行的步骤说明（人话）
  - 金额（高亮）
  - "确认 / 取消" 按钮
  - 倒计时（10 秒无操作默认取消）

## 用户偏好

- 设置中可配置守卫敏感度：保守 / 标准 / 激进
- 保守：所有点击都要确认
- 标准（默认）：金额 / 不可逆动作要确认
- 激进：仅金额 > 阈值要确认

## 永远不绕过

- 输密码 / 指纹这一步永远停下来等用户
- 即使 Skill 录制时记录了密码，回放时也不输入
- Agent 不存储 / 不上传任何密码

## 测试

- 模拟付款页面，断言守卫触发
- 模拟删除订单，断言守卫触发
- 模拟普通点击，断言守卫不误触

## 验收

- 五大场景中所有付款 / 不可逆点位 100% 命中守卫
- 误触率（普通点击被误判）< 1%
